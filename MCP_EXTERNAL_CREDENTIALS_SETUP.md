### What Needs Org UI Setup (Cannot Export via Metadata API)
⚠️ **These must be configured manually in your org:**

#### 1. Connected Apps (Setup → Apps → App Manager)
Create/Configure:
- **MCP Server Connection** (if using Anthropic MCP)
  - OAuth Scopes: `api`, `web`, `refresh_token`
  - Callback URL: Your MCP proxy callback endpoint
  
- **SAP Integration App** (if using OAuth)
  - Consumer Key: Generated by SAP
  - Consumer Secret: From SAP System

- **Weather API App** (OAuth or API Key)
  - OAuth URL endpoint
  - API Token (if using token auth)

#### 2. External Credentials (Setup → Security → External Credentials)
Create:
- **SAP_System_Credentials**
  - Credential Type: OAuth 2.0 or Service Token
  - Authentication Protocol: OAuth 2.0 / Client Credentials
  - Token Endpoint: `https://sap.example.com/oauth/token`
  - Scope: `ZBAPI_INVENTORY`

- **Weather_API_Credentials**
  - Credential Type: API Token / OAuth
  - Token Endpoint: Weather service provider

- **MCP_Server_Credentials** (if using authenticated MCP)
  - Credential Type: OAuth 2.0
  - Endpoint: Your MCP server auth endpoint

#### 3. Authentication Protocols (Setup → Security → Auth. Providers)
Link External Credentials to Auth Protocols:
```
Named Credential: SAP_Mock_API
  └─ External Credential: SAP_System_Credentials
      └─ Authentication Protocol: OAuth 2.0 (SAP)

Named Credential: Weather_Endpoint
  └─ External Credential: Weather_API_Credentials
      └─ Authentication Protocol: API Token (Weather)
```

## 🛡️ Security Best Practices

1. **Never commit secrets** - Use External Credentials (not hardcoded in flows)
2. **Use OAuth when possible** - More secure than API keys
3. **Set expiration** - External credential tokens should have TTL
4. **Audit access** - Monitor External Credential usage in Setup Audit Trail
5. **Limit scope** - Only grant necessary OAuth scopes
6. **Encrypt PII** - Ensure SAP data is encrypted in transit/rest
